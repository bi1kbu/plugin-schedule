# Admin Kit 样式复用说明

目录：`ui/src/styles/admin-kit/`

## 设计目标
- 将后台页面样式拆分为可复用模块，减少后续插件重复造样式。
- 保持当前插件后台视觉一致（卡片、表单、筛选、表格、分页、详情展开）。

## 文件分组与使用场景
1. `00-tokens.css`
- 场景：统一色板/边框/主色令牌。
- 要求：页面根节点带 `.page`，令牌在该节点生效。

2. `01-layout.css`
- 场景：页面骨架、标题区、提示消息、卡片容器、文本基础样式。
- 适合：任何包含「页面标题 + 多卡片」的控制台插件页。

3. `02-form-controls.css`
- 场景：网格表单布局、输入框、选择器、只读态、checkbox 组。
- 适合：创建/编辑/规则配置等表单页。

4. `03-actions-and-editors.css`
- 场景：按钮体系、关系选择器、chip 标签、选项编辑器。
- 适合：带批量关系操作、列表编辑的管理页。

5. `04-table-and-filter.css`
- 场景：筛选面板、筛选 pill、表格样式。
- 适合：台账列表、日志列表、检索页。

6. `05-detail-and-pagination.css`
- 场景：行展开明细、关系图标、分页区。
- 适合：可展开表格行和分页列表。

7. `90-responsive.css`
- 场景：移动端/窄屏降级布局。

8. `index.css`
- 场景：单入口引入，自动按顺序加载上述分组。

## 推荐接入方式
在页面 SFC 中使用：

```vue
<style scoped>
@import '../styles/admin-kit/index.css';
</style>
```

说明：
- 使用 `scoped` 时，样式仅作用于当前页面。
- 如果你需要跨多个页面共享同一外观，建议每个页面都引入同一 `index.css`。

## 最小类名约定
- 页面根容器：`.page`
- 内容卡片：`.card`
- 表单网格：`.grid` / `.grid.compact`
- 筛选区：`.filter-panel` + `.filter-row` + `.filter-pill`
- 列表区：`.table-wrap` + `table`
- 分页区：`.pager`

## 开发建议
- 新插件先复用现有类名，先搭结构再按需增量扩展。
- 若新增通用样式，优先补充到对应分组文件，避免散落在单页面。
- 若仅服务某一业务页面，不应写入通用 kit，避免污染复用语义。
