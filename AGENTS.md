# `plugin-schedule` 代理执行规范

## 语言规范
- 从现在起，所有沟通、记录、计划、备注统一使用中文。
- 新增文档与过程状态更新默认使用中文字段与中文描述。

## 计划唯一来源
- `implementation-plan.yaml` 是项目进度的唯一事实来源。
- 必须严格按顺序执行步骤（`M1` -> `M2` -> `M3` -> `M4` -> `M5` -> `M6`）。
- 任一时刻仅允许一个步骤为 `进行中`。

## 必须执行的步骤流程
1. 读取 `implementation-plan.yaml`。
2. 找到第一个状态为 `待开始` 的步骤。
3. 将该步骤状态改为 `进行中`。
4. 执行该步骤对应的实现工作。
5. 完成后将该步骤状态改为 `已完成`，并更新 `备注`。
6. 重新读取 `implementation-plan.yaml`。
7. 继续处理下一个 `待开始` 步骤。
8. 直到所有步骤都为 `已完成` 才停止。

## 状态管理规则
- 允许状态值：`待开始`、`进行中`、`阻塞`、`已完成`。
- 如遇阻塞，立即将状态改为 `阻塞`，并在 `备注` 写明阻塞原因。
- 阻塞解除后，将状态改回 `进行中` 并继续执行。
- 除非在 `implementation-plan.yaml` 明确调整，否则不得跳步。

## 更新纪律
- 每完成一个步骤，必须立即回写 `implementation-plan.yaml`。
- 未完成上一步状态落盘前，不得开始下一步。
- `备注` 必须简洁、可执行、可追踪。

## 调试方法
- 使用 `gradlew.bat haloServer` 运行开发环境，用户名和密码均为admin
- 使用 Chrome MCP 访问相应页面及相关操作，完成整体测试
